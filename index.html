Multidisciplinary Design Project. Rome wasn't built in a day.

Design a robot with sensory apparatus to explore a randomly constructed maze and to traverse between two designated points in the least possible amount of time

In case you're not familiar with The Little Prince, it's actually a French novella Le Petit Prince written by Antoine de Saint-Exupéry.

My contributions to this project are mainly on the following two core functions.

  • A robot controller to issue instructions. Our robot is like a toddler: it can walk around and sense the world but relies on the controller to tell it what to do. The controller is connected to the robot remotely via Transmission Control Protocol. Moreover, it provides a robot simulator (as shown above) that issues commands to a virtual robot, which allowed us to quickly test alternative algorithms before the physical robot was built.

  • An intelligent exploration algorithm based on Depth-first search. When the robot gets stuck in the maze, it needs to backtrack. It's easier to implement backtracking in software by just poping from the stack which in hardware, however, means the robot has to physically (costly) reverse along the previous path. As the maze gets explored, a shorter backtracking path may exist. Therefore, we used A* search algorithm to find such a path, which saved us tremendous time.

First Android™ Project. Naïve but practical.

Develop an Android application to allow patients to manage their appointments at ease

Patient Appointment Management System (PAMS) aims to create a hassle-free platform to foster better physician-patient communication. Its benefits are twofold:

  • Patients are automatically assigned available physicians. They can reschedule and receive updates on their appointments. Given that some treatments are partially ordered (like some ultrasound tests go before urine tests), PAMS prevents patients from jumping ahead treatments.

  • Physicians are able to view the medical records of a patient and schedule a referal appointment for a patient with another physician as the treatment procedure goes.

I worked on the login and signup functions as well as wrote a module to standardise communications with our backend server. Messages are sent to the server via HTTP requests in json format. Check out our PAMS-Android-Frontend and PAMS-Django-Backend. Of course, our video as well.

Deep Convolutional Neural Network. How AlphaGo beat Mr. Lee Sedol

Apply deep convolutional neural networks and autoencoders to recognise hand-written digits

This project is a mining of the MNIST database which contains images of hand-written digits in various shapes and orientations. We used 5,000 training images and 1,000 test images, eash of which has a 28x28 dimension. The experiment consists of two parts:

  • A deep convolutional neural network. We used two convolution layers of size 70 and 50, respectively, two mean subsampling layers, one fully connected layer and one softmax layer with 10 neurons as the output layer. The test accuracy was around 89%.

  • A stacked autoencoder. In an unsupervised fashion, we first trained two autoencoders of size 120 and 60, respectively, to extract features from the images. Then we trained a softmax output layer using a supervised learning algorithm. The test accuracy was around 93%.

Do check out our report as well as our Git repository.
